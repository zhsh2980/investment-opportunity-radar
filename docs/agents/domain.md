# 域文档

工程 skill 在探索本仓库时如何消费域文档。

## 探索代码前先读

- 根目录 **`CONTEXT.md`** — 项目术语表（本仓库为单上下文布局，没有 CONTEXT-MAP.md）
- **`docs/adr/`** — 阅读与将要改动的区域相关的 ADR

如果这些文件不存在，**静默继续**，不要提示缺失、不要主动建议创建。`/domain-modeling`（经 `/grill-with-docs` 触达）会在术语或决策真正敲定时惰性创建它们。

## 使用术语表的词汇

输出中提到域概念时（issue 标题、重构提案、假设、测试名），使用 `CONTEXT.md` 里定义的术语（如「简报」「要点」「机会类信息源」「破例推送」），不要漂移到术语表刻意避开的同义词。

需要的概念不在术语表里时，这本身是信号——要么你在发明项目不用的语言（重新考虑），要么存在真实缺口（记下来交给 `/domain-modeling`）。

## 显式指出 ADR 冲突

如果你的输出与现有 ADR 矛盾，明确说出来而不是悄悄覆盖：

> _与 ADR-0001（钉钉为唯一触达渠道）矛盾——但值得重新讨论，因为……_
