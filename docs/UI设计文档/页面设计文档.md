# 页面设计规范与验收标准 (Design System & Verification)

本文档总结了 "投资机会雷达" (Investment Opportunity Radar) 的 **Premium SaaS** 设计风格、核心视觉规范以及页面验收的技术标准。所有新页面开发和现有页面修改都必须遵循此文档。

---

## 1. 设计理念 (Design Philosophy)

*   **Premium (高级感)**: 拒绝廉价的色彩和复杂的装饰。使用细腻的阴影、圆角和留白来传递专业感。
*   **Clean Slate (极简)**: 以内容为核心。背景采用极浅的中性灰 (`Slate-50`)，内容承载于白色卡片 (`White`) 之上。
*   **Modern (现代感)**: 引入磨砂玻璃 (`Backdrop Blur`)、微交互和流畅的过渡动画。

---

## 2. 视觉规范 (Visual Standards)

### 2.1 色彩系统 (Color Palette)
我们严格使用 Tailwind CSS 的 **Slate (Neutral)** 和 **Indigo (Primary)** 色系。

| 用途 | 色值代码 (Tailwind Reference) | Hex | 说明 |
| :--- | :--- | :--- | :--- |
| **页面背景** | `bg-slate-50` | `#f8fafc` | **严禁使用深色/渐变背景** |
| **卡片背景** | `bg-white` | `#ffffff` | 必须纯白 |
| **主标题** | `text-slate-900` | `#0f172a` | 接近纯黑，深沉厚重 |
| **正文/副标题** | `text-slate-600` / `500` | `#475569` | 阅读舒适的灰色 |
| **品牌主色** | `text-indigo-600` | `#4f46e5` | 用于链接、激活状态、重点按钮 |
| **品牌背景** | `bg-indigo-50` | `#eef2ff` | 用于选中项背景、轻量徽章 |

### 2.2 排版 (Typography)
*   **字体**: 系统默认字体栈 (`Inter`, `-apple-system`, `PingFang SC`).
*   **层级**:
    *   **Page Title**: `text-2xl` or `text-3xl`, `font-bold`, `tracking-tight`.
    *   **Section Header**: `text-lg`, `font-semibold`.
    *   **Body**: `text-base` (16px), `leading-relaxed`.

### 2.3 质感 (Texture)
*   **边框**: 极其低调的边框 `border-slate-200`，用于界定卡片轮廓。
*   **阴影**: 扩散柔和的阴影 `shadow-sm` (常态) -> `shadow-md` (悬停)。
*   **圆角**: 统一使用 `rounded-lg` (12px) 或 `rounded-xl`。

## 3. 组件规范 (Component Standards)

### 3.1 核心容器卡片 (Card Component)
**已废弃**自定义类名 `.card-premium`。
必须直接使用 Tailwind Utility Classes 组合：
```html
<div class="bg-white rounded-lg border border-slate-200 shadow-sm p-6 hover:shadow-md transition-shadow">
    <!-- Content -->
</div>
```

### 3.2 状态徽章 (Status Badges)
使用 "Pill" 形状（圆角全满）：
*   **成功/机会**: `bg-green-50 text-green-700 ring-1 ring-green-600/20`
*   **中立/普通**: `bg-slate-100 text-slate-600`
*   **警告**: `bg-amber-50 text-amber-700 ring-1 ring-amber-600/20`

### 3.3 导航栏 (Sticky Navbar)
利用透明度和模糊效果增加层次感：
```css
.navbar-shell {
    background-color: rgba(255, 255, 255, 0.85);
    backdrop-filter: blur(12px);
    border-bottom: 1px solid var(--slate-200);
}
```

## 4. 技术验收标准 (Technical Verification Checklist)

在提交任何页面修改前，**必须**按顺序检查以下项目：

### ✅ 1. 样式加载检查 (Style Loading)
*   [ ] **Tailwind CDN**: 确认 `<head>` 中引入了 Tailwind CDN 脚本（解决 utility classes 无法被自定义 CSS 覆盖的问题）。
    ```html
    <script src="https://cdn.tailwindcss.com"></script>
    ```
*   [ ] **Cache Busting**: 确认 CSS 链接包含版本号参数（如 `?v=2`），强制刷新用户缓存。
    ```html
    <link rel="stylesheet" href="/static/app.css?v=2">
    ```

### ✅ 2. 背景色检查 (Background Verification)
*   [ ] **Reject Blue Gradient**: 检查 `<body>` 标签。
    *   **错误**: 带有 `linear-gradient` 或深蓝色背景。
    *   **正确**: 纯色背景 `bg-slate-50` (`#f8fafc`)。

### ✅ 3. 布局一致性 (Layout Consistency)
*   [ ] **Header**: 顶部导航栏是否固定 (`sticky`)？滚动时是否有磨砂效果？
*   [ ] **Centering**: 页面内容是否在 `container mx-auto` 中居中？
*   [ ] **Spacing**: 元素间距是否呼吸及合适（不仅促挤在一起，也没有过大的空白）？

### ✅ 4. 浏览器控制台 (Console Check)
*   [ ] 确认无 404 静态资源错误。
*   [ ] 确认 Tailwind 配置脚本无报错。

## 5. 快速纠错指南 (Troubleshooting)

| 现象 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| **页面样式全乱，全是原生 HTML** | CSS 404 或 MIME 类型错误 | 检查 Docker Nginx/FastAPI `StaticFiles` 配置，确认文件路径。 |
| **部分样式生效，但排版/颜色不对** | 缺少 Tailwind Utility Classes | 在 HTML `head` 中添加 Tailwind CDN Script。 |
| **修改了 CSS 但刷新没变化** | 浏览器强缓存 | 在 CSS 引用链接后加 `?v=timestamp` 或 `?v=n+1`。 |
| **背景一片死黑或深蓝** | 继承了旧版 CSS 或 Body Class | 检查 `base.html` 的 `body` 标签，移除旧的 `bg-gray-900` 类。 |
